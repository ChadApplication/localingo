## 2026-03-19 (3) AI Detection 관련 논의

**시간**: 심야
**주제**: Localingo 번역/요약 출력물의 AI Detection 이슈

### 배경

Localingo로 번역/요약된 텍스트를 학술 논문이나 문서에 활용할 때, AI Detection 도구(Turnitin, GPTZero, Originality.ai 등)에 감지될 수 있음.

### AI 번역 텍스트의 특징

- **Perplexity가 낮음** (예측 가능한 단어 선택)
- **Burstiness가 낮음** (문장 길이/복잡도가 균일)
- 이 두 지표가 AI Detection의 핵심. 번역은 원문이 존재하므로 "생성"과 성격이 다르지만 Detection 도구는 구분 못 함.

### 우회/최소화 방법 분석

| 방법 | 효과 | 윤리적 수준 |
|------|------|-----------|
| 후편집(Post-editing) — AI 번역 후 사람이 수정 | 높음 | 완전 정당 |
| 문체 변환 — 다른 톤/스타일로 재작성 | 중간 | 회색 |
| 혼합 전략 — 일부 AI, 일부 직접 | 중간 | 회색 |
| Perplexity 주입 — 비표준 단어/구조 삽입 | 낮음 | 비권장 |
| 워터마크 제거 — LLM 패턴 변경 | 어려움 | 비권장 |

### 합의: 학술적으로 정당한 접근

**"AI 사용을 숨기지 않고, 투명하게 밝히면서 품질을 보장"**

1. AI-assisted translation + human post-editing 워크플로우
2. 논문에 "translategemma로 번역 후 연구자가 검수" 명시

### Localingo에 반영할 기능 (Phase 향후)

| 기능 | 설명 |
|------|------|
| 후편집 모드 | 번역 결과를 직접 수정 가능한 편집 영역 |
| 수정 추적 | AI 원본 vs 사람 수정본 diff 표시 |
| AI 사용 명시문 자동 생성 | "Translated using translategemma, reviewed by author" |

## 2026-03-19 (4) 구현 완료 사항

**시간**: 새벽
**주제**: Phase 1-2 구현 + 추가 기능

### 완료 항목

1. **백엔드**: Ollama 연동, 번역/요약 API, SQLite 이력, 세션 CRUD
2. **프론트**: 인스턴트 번역 + 세션 카드 + 채팅형 번역 UI
3. **청킹**: 설정 가능 (XSmall 100자 ~ XLarge 5,000자), 기본 Medium

6. **UI 언어**: 한/영/중 전환, cookie 저장, 타겟 언어 연동

9. **세션**: 생성/삭제/수정/MD 다운로드
